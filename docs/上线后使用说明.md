# 上线后使用说明

这份说明面向日常实际使用，重点覆盖登录、记录、导入导出、备份和常见注意事项。

## 1. 登录使用

系统当前使用真实邮箱登录。

日常入口：
- 首页：部署后的正式网址
- 登录页：`/login`
- 报价生成器：`/quote-generator`

建议：
- 使用固定邮箱长期登录，不要频繁更换账号
- 管理员账号单独保留，避免误操作权限

## 2. 日常记录流程

推荐录入顺序：

1. 先写客户信息
2. 再写型号、港口、船公司、业务说明
3. 最后写跟进事项、反馈或待办

推荐格式：

```text
客户：客户姓名+公司名称
客户：客户姓名
客户：+公司名称
```

匿名示例：

```text
客户：A+B Company Ltd
型号：MODEL-01
港口：PORT A
备注：样品已确认，等待后续反馈
```

说明：
- `客户：客户姓名+公司名称` 会拆成“客户 + 公司”
- `客户：客户姓名` 会识别为客户
- `客户：+公司名称` 会识别为公司，不会生成客户

## 3. 识别规则使用建议

目前系统会重点识别：
- 客户
- 公司
- 港口
- 国家
- 型号
- 船公司
- 流程关键词

为了提高识别准确率，建议你尽量保持这些写法稳定：
- 客户行统一用 `客户：`
- 型号统一用 `型号：` 或 `品名：`
- 港口尽量写标准名称，如 `PORT A`、`PORT B`、`PORT C`
- 船公司尽量写标准简称，如 `MSC`、`CMA`、`COSCO`

## 4. 导入与导出

### Excel 导出

适合场景：
- 定期备份
- 月度整理
- 发给同事或自己线下分析

建议：
- 每周至少导出一次
- 重要阶段可以按月留档

### Excel 导入

适合场景：
- 从旧表格补录
- 批量恢复历史记录

建议：
- 导入前先确认账号正确
- 导入后抽查几条记录，确认识别结果正常

## 5. 数据备份建议

虽然系统现在已经使用 Supabase 持久化，但依然建议养成备份习惯。

推荐备份方式：

### 方式 A：应用内导出 Excel

这是最简单的日常备份方式。

建议频率：
- 每周一次
- 大批量更新前后各一次
- 月底固定导出归档

### 方式 B：Supabase 后台导出表数据

适合更正式的备份。

建议重点备份：
- `records`
- `tags`
- `profiles`

### 方式 C：双重备份

最推荐长期使用的方式：
- 平时用应用内 Excel 导出
- 每月再从 Supabase 后台做一次完整备份

## 6. 常见注意事项

### 不要依赖浏览器缓存

现在正式数据已经保存在 Supabase，不要再把浏览器本地缓存当成唯一数据来源。

### 不要把 `.env` 上传到 GitHub

环境变量应该只保留在：
- 本地 `.env`
- Vercel Environment Variables

### 如果识别不准怎么办

可以继续保持固定书写格式，然后逐步补规则。

目前最适合继续增强的方向有：
- 港口
- 客户跟进状态
- 记录结构化筛选

## 7. 上线后建议检查清单

每次大改后，建议手动检查：
- 能否登录
- 能否新增记录
- 能否编辑记录
- 能否删除记录
- 能否导出 Excel
- 报价生成器是否正常
- 刷新页面后数据是否仍存在

## 8. 建议的长期使用方式

如果你准备长期把它当工作系统使用，建议这样维护：
- 固定一个主账号
- 每周导出一次 Excel
- 每月做一次 Supabase 备份
- 识别规则按真实记录持续微调
- 上线前先做一次新增/编辑/导出检查
